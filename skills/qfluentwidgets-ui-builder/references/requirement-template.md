# Requirement Template

Copy this template before starting design.

## Input Sheet

1. Goal
- What problem this UI solves.
- Who uses it.

2. Screens
- List required pages.
- For each page, one-line responsibility.

3. Core interactions
- Main user actions.
- Success and failure behavior.

4. Data volume and constraints
- Data size class: small, medium, large.
- Need pagination: yes or no.
- Need async background tasks: yes or no.
- Platform constraints.

5. Scope
- In scope.
- Out of scope.

6. Acceptance
- Observable success criteria.
- Smoke test conditions.

## Output Lock

Do not move to architecture until all six sections have at least one concrete item.
